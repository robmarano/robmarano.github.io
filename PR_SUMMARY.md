# PR Summary: Week 9 Communication and Coordination & Distributed ZooKeeper Implementation

**Intended Changes**
* Author `courses/ece465/2026/weeks/week_09/notes_week_09.md` focusing on Communication (RPC/Message Passing) and Coordination (Election, Consensus) based on the van Steen & Tanenbaum textbook.
* Introduce an advanced Kubernetes-based architecture in `courses/ece465/2026/weeks/week_09/k8s_zk_template`.
* Implement a highly fault-tolerant Homogeneous MapReduce Image processing engine via Flask/SocketIO/Pillow.
* Rely on Apache ZooKeeper (`kazoo`) for leader election and ephemeral task distribution watches.
* Configure Kubernetes RBAC allowing the elected Master pod to label itself dynamically, routing the internal `master-service` DNS proxy instantaneously to the reigning leader.

**Implementation Details**
* **Notes**: Authored comprehensive markdown detailing Socket constraints, RPC Marshalling, Publisher/Subscriber topologies, Lamport Clock synchronization concepts, the Bully/Ring election algorithms, and finally mapping those concepts directly to Apache ZooKeeper theory.
* **Kubernetes Infrastructure**: Deployed `manifests` for a native ZooKeeper instance, a hostPath Persistence Volume mapped to Minikube, ServiceAccounts for Pod-relabeling, and a homogeneous Deployment with 3 initial identical replicas.
* **Master Controller**: Python threads use `kazoo.recipe.election` to race for an ephemeral lock. The winner automatically patches its K8s namespace labels with `role: master`. This flawlessly hooks into K8s internal DNS to serve the external websockets UI. 
* **ZooKeeper Coordinated MapReduce**:
    *   **Map Phase**: Master slices images vertically using `numpy`, scattering ephemeral `task` Znodes to `/jobs/histogram`. Workers claim them, execute `PIL` logic, and report local metrics arrays.
    *   **Barrier**: Master watches the ZTree structure until all arrays return, and accurately computes global normalized CDF metrics.
    *   **Reduce Phase**: CDF metric definitions scattered via new ephemeral `/jobs/equalization` tasks.
    *   **Fault Tolerance**: Required active minimum set to 5 Nodes. Blocks user traffic and broadcasts websocket warnings if cluster resources drop, resolving fully upon K8s redeployment/scaling.
    *   **Format Portability**: Expanded `core/image_processing.py` to seamlessly parse formats locally, opening robust `.tiff` file ingestion and reconstruction mapping natively.
* **Distributed UI Pipeline**:
    *   **Kubernetes RBAC Harvesting**: Expanded Master core API privileges (`k8s/rbac.yaml`) specifically for `pods/log` `get/watch` manipulation. The Master executes `urllib3` infinite HTTP request chunk iteration directly into K8s Control Plane. 
    *   **Robust Daemon Auto-Reconnect**: Embedded auto-reconnect logic capturing silently terminated HTTP log feeds natively generated by K8s Master.
    *   **Dashboard Visual Styling**: Completely reconstructed `index.html` frontend from simple centered bounds into a totally immersive Masonry UI Grid scaling massively utilizing deep CSS variables targeting `Courier New` simulated live mini-terminals.
* **Advanced Resilience & Testing (Latest Additions)**:
    *   **OS Thread Offloading**: Mitigated heavy NumPy CPU starvation (e.g. 75MB TIFF slices) from crashing the `eventlet` Kazoo heartbeat threads by routing synchronous blockades through `eventlet.tpool.execute()`.
    *   **Graceful ZooKeeper Reconnects**: Enveloped all cluster MapReduce instructions (create, get, ensure_path) natively in `zk.retry()` fault-tolerant encapsulations to survive momentary Minikube socket-read timeouts cleanly.
    *   **End-to-End Testing**: Authored PyTest infrastructure (`tests/test_image_processing.py`, `tests/test_e2e_pipeline.py`) validating geometry splitting and invoking the full Kubernetes Socket.IO stack asynchronously against high-res binary testing assets.
