Catalog Description:

Introduction to the design of computers and computer systems. Topics include: integer and floating-point representations and operations: ALU design; von Neumann and Harvard architectures; accumulator, general purpose register and stack-based processor design; RISC and CISC architectures; addressing modes; vector operations; microprogrammed and hard-wired controllers; machine language and assembly language programming; static and dynamics memory operation, timing and interfacing; cache; virtual memory; I/O systems: bus design and data transfer, DMA; interrupts and interrupt handling, polling; disk operation and organization; pipelined processor design. The course has a substantial project component that includes assembly language programming and the design and construction of systems that contain microcontrollers, programmable logic, and a variety of I/O devices.

Goals and Outcomes:

  • At the completion of this course, students will be able to satisfy the following:
  • Define computer architecture
  • Evaluate the performance of a computer system
  • Explain the instruction set architecture and its importance in the design of computer systems
  • Design combinational and sequential digital logic circuits
  • Explain the design of the arithmetic logic unit (ALU) and the processor, datapath and control
  • Provide the values of control line to implement instructions
  • Describe the organization of the central processing unit (CPU), input/output (IO) devices, and memory hierarchy
  • Show how memory caches and virtual memory work
  • Evaluate the performance of a pipeline
  • Evaluate different IO systems
  • Classify multiprocessor systems

  • Brainstorm your final project in this class. You will have two options: a hardware only solution either physically built with logic chips or simulated on Verilog software. The theme of the project is to create a computer of your choice with a minimum of 4 bits and no more than 8 bits, given the complexity expected from this course. You will design the entire ISA, the data path, the control path, the ALU and the Controller units, along with the support for integer processing. How would you handle floating point? If you like, you can choose an alternative path and write a functional LEGv8 simulator that shows the states of each register and how the memory works, along with writing the control path, data path, ALU, Controller.
  • You will work in groups of two.
  • For next class, submit via Teams a one-page write-up of what you intend to pursue for your project. You and your partner hand in the same write-up but in your own respective Team’s account in Team’s Assignments.
