## Problem Set

### Part 1: Advanced Assembly & Hardware Profiling

**1. The Cost of Hardware Execution**

**Goal**: Understand why instruction count in a software emulator like SPIM does not natively equate to physical hardware latency on silicon.

In our analysis of the Quake III Arena *Fast Inverse Square Root* algorithm, we replaced a standard 2-instruction IEEE-754 sequence (`sqrt.s` followed by `div.s`) with a 14-instruction sequence utilizing integer and bitwise manipulation (`srl`, `sub`, `mul.s`). 

*   **A.** If you run both algorithms sequentially 100,000 times in the SPIM software emulator, the 2-instruction standard approach finishes almost 3x faster. Explain why the *Simulator Illusion* misrepresents true hardware performance here.
*   **B.** Explain the engineering and hardware justification behind Quake developers choosing to execute 14 separate instructions instead of relying on the single, highly accurate `div.s` instruction on 1999 silicon (like the MIPS R4000).

**2. Exception Handling Architecture**

**Goal**: Identify how the MIPS ISA mechanically handles fatal hardware faults or OS-level interrupts during runtime execution.

When an unaligned memory access (Address Error) or an arithmetic anomaly (Overflow) occurs, standard user-space execution is halted.
*   Describe the architectural role of **Coprocessor 0 (CP0)** and the `.ktext` memory segment in capturing and resolving the fault. 
*   What explicitly happens to the user program's `PC` (Program Counter) when SPIM's `exceptions.s` kernel handler takes over?

### Part 2: Final Project & Midterm Readiness

**3. ISA Design and Bit Constraints**

**Goal**: Evaluate the architectural layout of custom instruction widths for your impending Final Project ISA.

For your Final Project, you are structurally required to design an Instruction Set Architecture (ISA). Assume you are building a strict 32-bit width instruction format similar to MIPS.

You decide you want your CPU to support immediately loading massively large constant numbers directly inside an I-Type instruction. To do this, you assign **24 bits** out of the 32 purely to the "Immediate" field. 
*   Mathematically, how does this 24-bit immediate field explicitly constrain the remaining bits available for your Opcode and Register addresses? 
*   What is the architectural consequence of this decision on the total number of unique registers your CPU can physically wire up?

**4. MIPS Dataflow & Recursion**

**Goal**: Solidify execution chains, Stack Pointer (`$sp`) usage, and Return Address (`$ra`) logic for the midterm.

A student writes a MIPS procedure that recursively calls itself to calculate a Factorial. However, before executing the `jal` instruction to jump back into its own loop, the student completely forgets to use the `$sp` (Stack Pointer). 

*   What specific mechanical hardware state is catastrophically overwritten when the second `jal` executes? 
*   Explain precisely how this omission destroys the `jr $ra` execution chain when the recursion finally hits its base case and attempts to unfold back to the `main` procedure.
