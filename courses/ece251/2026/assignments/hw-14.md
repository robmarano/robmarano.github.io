## Problem Set

### Part 1: Error Detection and Correction

**1. Hamming Code Parity and Syndrome Decoding**

a) What is the minimum number of parity bits required to protect a 128-bit word using the SEC/DED (Single Error Correction / Double Error Detection) code?

b) Consider a single error correcting (SEC) code that protects 8-bit data words with 4 parity bits (yielding a 12-bit total word). If the memory controller reads the hexadecimal value `0x375` from DRAM, is there an error? If so, identify the exact bit position that flipped and provide the corrected hexadecimal value. Show your overlapping parity equations and syndrome derivation for full credit.

### Part 2: Virtual Memory Tracing

**2. TLB, Page Table, and Page Faults**

Assume a Virtual Memory system with 4 KB pages. The hardware possesses a 4-entry fully associative TLB with True LRU replacement. The current state of the TLB and Page Table is given below:

*Current TLB State:*

| Valid | Tag (VPN) | Physical Page Number (PPN) | LRU Age (Higher is older) |
|:---:|:---:|:---:|:---:|
| 1 | 11 | 12 | 4 |
| 1 | 7  | 4  | 1 |
| 1 | 3  | 6  | 3 |
| 0 | 4  | 9  | 7 |

*Current Page Table State (Partial Snapshot):*

| Index (VPN) | Valid | PPN or Location |
|:---:|:---:|:---:|
| 0 | 1 | 5 |
| 1 | 0 | Disk |
| 2 | 0 | Disk |
| 3 | 1 | 6 |
| 8 | 0 | Disk |

The CPU executes a sequence of memory accesses at the following decimal Virtual Addresses: `4669`, `2227`, `13916`, and `34587`.
For *each* access sequentially, determine and state:
a) The Virtual Page Number (VPN).
b) Whether the access results in a TLB Hit or TLB Miss.
c) Whether the access results in a Page Table Hit or a Page Fault.
d) If the access requires loading a new mapping into the TLB, which specific TLB entry does it overwrite or populate?

### Part 3: Page Table Sizing

**3. Single-Level vs Multi-Level Overhead**

Consider a 64-bit operating system utilizing a sparse 43-bit Virtual Address space, backed by 16 GB of Physical DRAM. The system uses a standard 4 KB page size and each Page Table Entry (PTE) consumes 4 bytes.

a) If the OS uses a strict single-level page table, how many total Page Table Entries must it maintain?
b) How much physical memory (in GB) is consumed strictly by this single-level page table?
c) What percentage of the total installed 16 GB DRAM is consumed by the page table? Explain briefly why this result necessitates the use of Multi-Level Page Tables in modern 64-bit architectures.
